Support Ticket

Structured support intake for incidents and operational issues. Receipted response.

Severity Levels

  • SEV-1 — verification-blocking outage
  • SEV-2 — degraded proof operations
  • SEV-3 — non-blocking defect or request

Submit Ticket

Required Evidence

  • Boundary — service + environment
  • Time — timestamp in UTC
  • Evidence — digest IDs or run IDs
  • Impact — who is blocked

Security Reporting
  • Priority — critical if active exploit path
  • Minimum evidence — repro + impact + logs
  • Reference — Trust model

Submission Routes
  • GET docs — this page
  • POST endpoint — /support/ticket
  • Status tracking — support.vaultmesh.org
