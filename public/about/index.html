Verification You Can Carry

VaultMesh builds infrastructure where claims are machine-verifiable, evidence is portable, and trust is earned through cryptographic proof — not promises.

The problem

Regulated systems depend on compliance reports that are written once and never verified. Audits are periodic. Evidence is locked in vendor dashboards. When something goes wrong, reconstructing truth is expensive, slow, and often impossible.

VaultMesh exists because proof should be continuous, portable, and independently verifiable — not a quarterly PDF.

Principles

  • State continuity is monotonic — append-only, no rewrites
  • Verification works offline — no phone-home required
  • Receipts are portable — export, ingest, dispute
  • Distrust is expected — adversarial validation by default
  • Minimal surface — complexity is the enemy of verification

How it works

  • Hash-chained ledgers (append-only receipts)
  • Ed25519 signatures (detached, offline verifiable)
  • Equivocation detection (fork and replay awareness)
  • Deterministic builds (reproducible artifact assembly)
  • Portable evidence bundles (export / ingest / dispute packs)

Trust boundaries

VaultMesh does not ask you to trust VaultMesh. Every claim is backed by a signed artifact that you can verify independently. The verification tools are open. The receipts are portable. If we disappear tomorrow, your proof still works. 

trust_model:
  signing: Ed25519 (detached)
  chains: SHA-256 (append-only)
  anchors: RFC 3161, ETH, BTC (optional)
  verification: offline, no vendor dependency
  evidence: portable bundles w/ dispute path
