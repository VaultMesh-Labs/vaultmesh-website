Verify a bundle.

Verification is designed to work offline. If you can unzip it, you can verify it.

Get a Proof Pack Trust model

What you need

  • The bundle zip (bundle.zip)
  • The verifier script (for example VERIFY_ALL.sh)
  • A standard shell + hash tools (no service dependencies)

3-step verification

  1. Unzip the bundle.
  2. Run the verifier script.
  3. Read the markers: green means verified; red means drift or mismatch.

What "good" looks like

  • Hashes match.
  • Receipts chain is continuous.
  • Verifier exits 0 and emits explicit success markers.

Failure modes (expected)

  • Missing files → bundle incomplete.
  • Hash mismatch → content changed.
  • Chain break → continuity violated.
