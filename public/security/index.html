Security Disclosure

Evidence first. Impact scoped. Responsible process.

Reporting a vulnerability

If you believe you have found a security vulnerability in VaultMesh infrastructure, please report it through the structured disclosure process below.

  • Email: security@vaultmesh.org
  • Encryption: Ed25519 verifying key at /security/verify.pub (raw 32 bytes)
  • Response target: acknowledgment within 48 hours

What to include

  • Reproducible steps (commands, URLs, inputs)
  • Impact scope (what is affected, severity estimate)
  • Evidence artifacts (logs, screenshots, proof of concept)
  • Your contact information for follow-up

Reports without reproducible evidence will be acknowledged but may not be actionable. The more specific the report, the faster the response.

Disclosure process

1. Report received → acknowledgment (48h target)
2. Triage → severity classification + evidence review
3. Fix development → patch prepared, regression tested
4. Coordinated disclosure → reporter notified before public
5. Public advisory → receipted, signed, append-only record

Scope

In scope:

  • VaultMesh proof chain integrity (hash chains, signatures, anchors)
  • State continuity violations (equivocation, replay, tampering)
  • Infrastructure vulnerabilities (edge, origin, bastion hosts)
  • Build pipeline compromise (deterministic build bypass)

Out of scope:

  • Social engineering against personnel
  • Denial of service (volumetric)
  • Issues in third-party dependencies (report upstream)
